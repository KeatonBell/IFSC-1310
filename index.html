Thoughts on Progressive Enhancement

Progressive enhancement seems very intuitive and efficient. With a good HTML file that has no errors you always have a website that is functional. If the CSS and Java Script fail users will still be able to use the website. This is very important, because if the HTML has errors, then it will all fall apart from there.

I also agree that focusing on creating a website that is appealing to the eye with CSS primarily is a goo practice as well. CSS is much less of a strain on the computer which is very beneficial for older devices. Web developers must remember that technology advances at a rate that some people can’t keep up with, and still have iPhone 5’s or still running a windows 7 computer. This need to be thought about before adding thousands of lines of Java Script that will take 5 minutes to compute correctly or not run right at all.

I do think Java Script is a very essential too to web development. It allows to do more than just animations and cool effects. It is what thousands of companies use for data collection, and algorithms that can help their company tremendously. So, at the end of the day, it all just a balance, but a house is built from the ground up. Just like any website created should be as well.
